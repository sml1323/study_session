# Setup — First Run and Bootstrap

Run on the first invocation of the skill, when `~/study-journal/` does not exist or is incomplete.

## Step 1: Verify directory structure

Expected layout:

```
~/study-journal/
├── README.md
├── .gitignore
├── books.yml
├── books/
│   └── <book-slug>/
│       ├── _index.md
│       └── ch-NN-<title-slug>.md
├── evergreen/
│   └── _index.md
├── outputs/
│   └── _index.md
└── standalone/
    └── problems/
```

Bootstrap order:
1. Create `~/study-journal/` if missing
2. Create subdirectories
3. Write `README.md` (overview, command list, conventions)
4. Write `.gitignore` (book PDFs are excluded — fair use is for personal learning, not redistribution)
5. Write `books.yml` (initially empty or with detected books)
6. Write subdirectory `_index.md` files

Use `scripts/init.sh` to do all of this in one call.

## Step 2: .gitignore

```gitignore
# Book files — fair use for personal learning only, never push
*.epub
*.pdf
books/*/*.pdf
books/*/*.epub

# Local-only
.DS_Store
.obsidian/workspace*
```

## Step 3: README.md

A small README so the user remembers what's in here:

```markdown
# Study Journal

Personal book learning notes generated by the `study-session` Claude Code skill.

## Structure
- `books/` — chapter-by-chapter notes, one file per chapter
- `evergreen/` — atomic principles extracted across chapters
- `outputs/` — composed artifacts (weekly reviews, exam prep)
- `standalone/` — ad-hoc problems / arguments not tied to a book chapter
- `books.yml` — book metadata + progress

## Commands
- `/study-session` — start or resume a session
- Natural language: "ARQ Ch.5 같이 보자", "오늘 학습", "내 노트 검토", "복습 퀴즈"

## Convention
- Append-only chapter notes (don't edit prior session entries)
- Phase 3 (calibrate) runs as the opening of the *next* session by default — the cross-session gap is the delay. Same-sitting calibrate is opt-in only and requires `now - phase_2_ended_at ≥ 30 min`. See `references/calibration.md` for rationale.
- Forms are auto-filled by the skill from session traces, not manually
```

## Step 4: books.yml bootstrap

Detect books in `~/wiki/tmp_books/` (or wherever the user keeps them — ask).

For each detected book:
- Generate a slug (e.g., `arq`, `polya`, `feynman-vol-1`)
- Detect format (PDF, EPUB)
- For EPUB: must convert to PDF for skill use (Read tool reads PDF)
- Capture path
- Set `type` if recognizable; else mark `pending_classification: true`

See `assets/books.yml.template` for the schema.

## Step 5: EPUB conversion

If any book is EPUB, attempt conversion to PDF.

### Tool detection

```bash
which pandoc        # check for pandoc
which ebook-convert # check for Calibre's CLI
```

### If both missing

Surface to user, do not auto-install:

```
EPUB → PDF 변환에 pandoc 또는 Calibre가 필요해. 둘 중 하나 설치해줘:

# Option A — pandoc (가벼움)
brew install pandoc

# Option B — Calibre (GUI 포함, 변환 품질 좋음)
brew install --cask calibre

설치 후 다시 호출하면 변환 진행할게.
```

### If pandoc available

```bash
pandoc input.epub -o output.pdf
```

Pandoc EPUB → PDF requires LaTeX backend (`brew install --cask basictex` if missing). If LaTeX missing, fall back to:

```bash
pandoc input.epub -o output.html
# Then user opens in browser and prints to PDF
```

Or use Calibre.

### If Calibre available (preferred for EPUB → PDF)

```bash
ebook-convert input.epub output.pdf
```

Calibre handles EPUB → PDF natively, no LaTeX needed.

### Verify conversion

After conversion:
- Check file exists
- Check file is non-empty
- Try Read tool on first 5 pages to verify it's text-extractable (scanned PDFs without OCR will fail)

If verification fails, surface the issue and the suggested fix (run OCR via `ocrmypdf input.pdf output.pdf` if scanned).

## Step 6: First book classification

For each book in books.yml without a `type`:

1. Read the table of contents (first 5-10 pages of the PDF)
2. Apply heuristics from `references/book-types.md`
3. Surface guess to user with one-sentence justification
4. User confirms or overrides

Update books.yml with the type.

## Step 7: First session prompt

After setup is complete:

> Setup 완료. 첫 세션 시작할까?
>
> books.yml에 등록된 책: ARQ (methodology), Polya Part I (methodology), Polya Part II (reference), Polya Part III (problem-driven)
>
> 무엇부터?
> - ARQ Ch.1
> - Polya Part I 시작
> - 다른 챕터 지정

## Setup failure modes

### Books in unexpected location

If `~/wiki/tmp_books/` doesn't exist, ask user where books are. Don't assume.

### EPUB conversion fails

Don't block the session. Offer:
1. Skip this book for now
2. User pastes chapter text manually each session (works for any format)
3. User converts manually and re-runs setup

### Existing study-journal/ but corrupt

If `~/study-journal/` exists but `books.yml` is missing or invalid:

1. Don't overwrite chapter notes (those are user data)
2. Reconstruct books.yml from `books/<slug>/_index.md` files if possible
3. Otherwise, ask user to confirm before any reset

### User changed book paths

If `books.yml` references a path that no longer exists, surface and ask. Don't silently re-bootstrap.

## What the user sees during setup

Be brief. Don't narrate every directory creation. Summary message:

> Setup 진행:
> - `~/study-journal/` 생성
> - books.yml 부트스트랩
> - ARQ.epub 발견 → PDF 변환 (pandoc 사용, 30초)
> - Polya.pdf 발견 → 그대로 등록
>
> 완료. 첫 세션 시작할까?

## After first run

Subsequent invocations skip setup. Only re-run setup steps that detect missing infrastructure (e.g., new book added, EPUB conversion needed for new book).
